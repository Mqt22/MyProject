Documented test cases

All functional test cases for the Visual Cryptography & Steganography Toolkit prototype.
Based on evaluation methodology from: Li et al., "Robust Image Steganography via Color Conversion," IEEE TCSVT 2025.

Feature 01 — LSB Steganography
TC-001
Encode & exact decode
Embed "Hello World" into any PNG. Decode must return exact string with 0 errors.
✓ Auto-verifiable
TC-002
Visual imperceptibility
PSNR of stego vs cover must exceed 30 dB. Expected: ~39 dB for short messages.
✓ Auto-verifiable
TC-003
Capacity limit warning
Message exceeding pixel capacity should trigger a visual warning, not silent truncation.
✓ Auto-verifiable
TC-004
Empty message handling
Embedding an empty string should produce a stego image with only the null terminator.
✓ Auto-verifiable
TC-005
Special characters
Unicode, emoji, and newline characters must encode/decode correctly via UTF-8.
⧖ Manual verify
Feature 02 — Visual Cryptography
TC-006
Shares appear random
Both Share A and Share B must be visually indistinguishable from random noise independently.
⧖ Manual verify
TC-007
XOR overlay reveals secret
Overlaying Share A XOR Share B must produce a recognizable reconstruction of the original.
✓ Auto-verifiable
TC-008
Deterministic shares
Different inputs must produce different shares. Same input always → same Share A (deterministic).
⧖ Manual verify
Feature 03 — PSNR / SSIM Metrics
TC-009
Identical images → ∞ PSNR
Uploading the same image twice must show PSNR = ∞ (MSE = 0) and SSIM = 1.000.
✓ Auto-verifiable
TC-010
Noisy image → lower PSNR
After Gaussian noise attack, PSNR should drop inversely with noise variance (σ²=25 → ~20–25 dB).
✓ Auto-verifiable
TC-011
Stego scores > 30 dB
LSB-encoded stego image must score above 30 dB PSNR vs original (matches Table I in paper).
✓ Auto-verifiable
Features 04–05 — De-colorize & Noise Attack
TC-012
Grayscale pixel range
All pixels in de-colorized output must have R=G=B (valid grayscale), values in [0, 255].
✓ Auto-verifiable
TC-013
Luminance formula accuracy
Pixel (255,0,0) → luminance = 76.245 ≈ 76. Pixel (0,255,0) → 149.7 ≈ 150.
✓ Auto-verifiable
TC-014
Gaussian noise distribution
Mean of pixel differences between original and noisy must be ≈ 0 (symmetric noise).
✓ Auto-verifiable
